# Product Reset Checklist

_Last updated: March 15, 2026_

## Before coding

- Confirm the primary product story is `bring in a book and listen`
- Confirm community is secondary, not primary
- Confirm first-run language replacements for:
  - edition
  - circle
  - moment
  - studio
- Identify the single primary CTA for home
- Identify the single primary CTA for the book page
- Identify which advanced controls will be hidden by default

## Home review

- Can a new user tell what the product is in one sentence?
- Is there exactly one dominant CTA?
- Is sample content easier to start than community exploration?
- Are backend/system concepts hidden or demoted?

## Import review

- Is there a clear source chooser?
- Are working import paths above roadmap content?
- Does the user know what works today?
- Can the user reach playback with minimal decisions?

## Book and player review

- Is `Play` or `Resume` obvious?
- Are progress and chapters visible?
- Are speed, timer, and bookmark easy to find?
- Are adaptive controls present but not overwhelming?

## Community review

- Can users ignore community completely and still succeed?
- Are community labels plain-language?
- Does community feel optional and inviting rather than required?

## Visual review

- Are primary actions high-contrast?
- Are there too many cards above the fold?
- Is the screen dominated by the thing the user cares about most?
- Are related actions grouped together?

## Validation

- Walk the sample-book path in a real browser
- Walk the text-import path in a real browser
- Walk the personal-audio path in a real browser
- Walk the return-and-resume path in a real browser
- Record the remaining top 5 confusion points
